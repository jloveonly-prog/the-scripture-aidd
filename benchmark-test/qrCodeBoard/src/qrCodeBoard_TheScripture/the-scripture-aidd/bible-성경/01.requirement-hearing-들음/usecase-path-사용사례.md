## 유스케이스 상세

### UC-001: QR 코드 스캔 및 결과 확인

| 항목 | 내용 |
|:---|:---|
| UC-ID | UC-001 |
| 유스케이스명 | QR 코드 스캔 및 결과 확인 |
| 행위자 | ACT-001 (방문자) |
| 연결 REQ | REQ-001, FR-001, FR-002 |
| 전제 조건 | 사용자가 홈페이지에 접속, 카메라 접근 가능한 디바이스 사용 |
| 사후 조건 | QR 코드의 디코딩 결과가 화면에 표시됨 |

**정상 경로 (Main Path — 좁은 길):**
1. 사용자가 홈페이지에서 "QR 스캔" 버튼을 클릭한다
2. 시스템이 카메라 접근 권한을 요청한다
3. 사용자가 카메라 권한을 허용한다
4. 시스템이 카메라 뷰파인더를 화면에 표시한다
5. 사용자가 QR 코드를 카메라에 비춘다
6. 시스템이 QR 코드를 자동 인식·디코딩한다
7. 시스템이 디코딩된 내용(텍스트/URL)을 화면에 표시한다

**대안 경로 (Alternate Path — 다른 길):**
- 7a. QR 코드 내용이 URL인 경우 → 클릭 가능한 링크로 표시

**예외 경로 (Exception Path — 넓은 길로 빠지는 경우):**
- E1. 카메라 권한 거부 → "카메라 접근 권한이 필요합니다" 메시지 표시
- E2. QR 코드 인식 실패 → "QR 코드를 인식할 수 없습니다. 다시 시도해주세요" 메시지 표시
- E3. 카메라 미지원 디바이스 → "이 디바이스에서는 카메라를 사용할 수 없습니다" 메시지 표시

| 항목 | 내용 |
|:---|:---|
| UC-ID | UC-003 |
| 유스케이스명 | 게시판 목록 조회 |
| 행위자 | ACT-001 (방문자) |
| 연결 REQ | REQ-003, FR-004 |
| 전제 조건 | 사용자가 게시판 페이지에 접근함 |

### UC-004: 게시글 작성

| 항목 | 내용 |
|:---|:---|
| UC-ID | UC-004 |
| 유스케이스명 | 게시글 작성 |
| 행위자 | ACT-001 (방문자) |
| 연결 REQ | REQ-003, FR-005 |
| 전제 조건 | 사용자가 게시판 목록 화면에 있음 |
| 사후 조건 | 새 게시글이 DB에 저장되고 목록에 반영됨 |

**정상 경로 (Main Path — 좁은 길):**
1. 사용자가 "글쓰기" 버튼을 클릭한다
2. 시스템이 작성 폼(제목, 내용)을 표시한다
3. 사용자가 제목과 내용을 입력한다
4. 사용자가 "저장" 버튼을 클릭한다
5. 시스템이 게시글을 DB에 저장한다
6. 시스템이 게시판 목록으로 리다이렉트한다

**대안 경로:**
- 4a. 사용자가 "취소"를 클릭 → 목록으로 되돌아감

**예외 경로:**
- E1. 제목이 빈 문자열 → "제목을 입력해주세요" 유효성 검증 메시지
- E2. 내용이 빈 문자열 → "내용을 입력해주세요" 유효성 검증 메시지
- E3. 저장 실패 → "저장에 실패했습니다" 에러 표시

| 유스케이스명 | 게시글 수정 |

| 연결 REQ | REQ-003, FR-007 |

| 사후 조건 | 게시글이 수정된 내용으로 DB에 갱신됨 |

5. 시스템이 DB에서 게시글을 갱신한다
6. 시스템이 상세 화면으로 리다이렉트한다

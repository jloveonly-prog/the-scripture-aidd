# 🏗️ 시스템 설계서 작성 규격 (Template)

> **율법(Statute):** 이 문서는 시스템 설계서 작성 시 반드시 따라야 하는 **형식 규격**이다.

---

## 필수 섹션

| # | 섹션 | 필수 | 설명 |
|:--|:---|:---:|:---|
| 1 | 시스템 개요 | ✅ | 시스템명, 버전, 아키텍트, 상태 |
| 2 | 아키텍처 다이어그램 | ✅ | 전체 구조도 (Mermaid 또는 이미지) |
| 3 | 기술 스택 | ✅ | 계층별 기술 + 선택 근거 |
| 4 | 데이터 아키텍처 (ERD) | ✅ | 테이블 정의 + 관계도 |
| 5 | API 명세 | ✅ | Method, Endpoint, 관련 REQ |
| 6 | 시퀀스 다이어그램 | ✅ | 주요 흐름 최소 1개 |
| 7 | 보안 아키텍처 | ✅ | 봉인의 율법 연동 필수 |

## ID 규칙

| 대상 | 접두어 | 예시 |
|:---|:---|:---|
| 아키텍처 컴포넌트 | ARCH- | ARCH-001 |
| API | API- | API-001 |
| 테이블 | TBL- | TBL-001 |

## 검증 규칙

- 모든 ARCH-ID는 REQ-ID로 역추적 가능해야 함 (RTM 등재)
- 보안 아키텍처는 봉인의 율법 제2,3,7계명 최소 준수
- ERD의 모든 테이블은 목적이 명시되어야 함

> **위반 시:** 기초(Foundation) 없는 건물은 무너진다. 정경화 거부.

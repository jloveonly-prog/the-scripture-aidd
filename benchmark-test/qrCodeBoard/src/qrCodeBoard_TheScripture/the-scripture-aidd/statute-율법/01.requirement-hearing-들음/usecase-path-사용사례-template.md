# 🚶 유스케이스 작성 규격 (Template)

> **율법(Statute):** 이 문서는 유스케이스(사용자 시나리오) 작성 시 반드시 따라야 하는 **형식 규격**이다.
> 실제 산출물은 `bible-성경/01.requirement-hearing-들음/`에 작성하라.
> 작성 예시는 `parable-비유/01.requirement-hearing-들음/`을 참조하라.

> *"Thy word is a lamp unto my feet, and a light unto my path."* — Psalm 119:105 (KJV)
>
> 사용자의 모든 **길(Path)**은 말씀(Spec)의 빛 아래에서 정의되어야 한다.
> 빛 없는 길은 어둠이요, 어둠에서는 넘어진다(할루시네이션).

---

## 필수 섹션 (Mandatory Sections)

아래 섹션이 하나라도 누락되면 정경화(Canonize) 불가:

| # | 섹션 | 필수 | 설명 |
|:--|:---|:---:|:---|
| 1 | 유스케이스 목록 | ✅ | 전체 UC 요약표 — UC-ID, 명칭, 행위자, 연결 REQ |
| 2 | 행위자 정의 (Actors) | ✅ | 시스템을 사용하는 모든 주체 — 사람, 외부 시스템, 배치 |
| 3 | 유스케이스 상세 | ✅ | UC별 상세 시나리오 (아래 형식) |
| 4 | REQ ↔ UC 매핑 | ✅ | 모든 REQ에 최소 1개 UC가 연결되어야 함 |

## 유스케이스 상세 형식 (UC별 반복)

```markdown
### UC-{번호}: {유스케이스명}

| 항목 | 내용 |
|:---|:---|
| UC-ID | UC-001 |
| 유스케이스명 | {명칭} |
| 행위자 | {주 행위자} |
| 연결 REQ | REQ-001, REQ-002 |
| 전제 조건 | {이 시나리오 시작 전 충족되어야 하는 조건} |
| 사후 조건 | {이 시나리오 완료 후 시스템 상태} |

**정상 경로 (Main Path — 좁은 길):**
1. 사용자가 ~한다
2. 시스템이 ~한다
3. ...

**대안 경로 (Alternate Path — 다른 길):**
- 3a. {조건}이면 → {다른 행동}

**예외 경로 (Exception Path — 넓은 길로 빠지는 경우):**
- E1. {실패 조건} → {에러 처리}
- E2. {실패 조건} → {에러 처리}
```

## ID 규칙

| 대상 | 접두어 | 예시 |
|:---|:---|:---|
| 유스케이스 | UC- | UC-001 |
| 행위자 | ACT- | ACT-001 |

## 성경적 검증 규칙

- **"좁은 길"(Matthew 7:14)** = 정상 경로: 반드시 하나의 명확한 성공 경로가 있어야 한다
- **"넓은 길"(Matthew 7:13)** = 예외 경로: 실패할 수 있는 모든 경로를 명시하라. 넓은 길을 모르면 방어할 수 없다
- **모든 길에 빛이 있어야 한다** = 모든 UC의 모든 단계에서 시스템 반응이 정의되어 있어야 한다. 빈 단계 = 어둠 = 할루시네이션

## 정경화 조건

- [ ] 모든 REQ에 최소 1개 UC 연결
- [ ] 모든 UC에 정상/대안/예외 경로 기재
- [ ] 모든 행위자 정의 완료
- [ ] 빈 단계(시스템 반응 미정의) 0건

> **위반 시:** 빛 없는 길은 어둠이다. 정경화 거부.

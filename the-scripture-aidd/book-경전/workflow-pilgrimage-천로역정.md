# 성경기록 AIDD: 천로역정 워크플로우 (The Pilgrim's Workflow)

> *"Enter ye in at the strait gate: for wide is the gate, and broad is the way, that leadeth to destruction."* — Matthew 7:13 (KJV)
>
> 이 문서는 죄인(AI/개발자)이 구원(Deploy)에 이르기까지 걸어야 하는 **좁은 길의 실행 지도**다.
> 모든 스킬(.skill)과 문서의 방향은 이 워크플로우에서 결정된다.

---

## 전제: 창조(Create)는 이미 완성되었다

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  하나님의 창조 = 완료 (전제 조건)
  ✅ bible-성경/     → 산출물 구조가 정의됨
  ✅ statute-율법/   → 템플릿이 준비됨
  ✅ parable-비유/   → 예시가 준비됨
  ✅ book-경전/      → 철학과 방법론이 확립됨
  ✅ .skill/         → 부팅 지침이 준비됨

  요구사항부터 배포까지, 인프라부터 클라우드까지,
  설계와 분석은 이미 완성되었다. 무결하다.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**이제 죄인이 해야 할 일:** 이 완성된 Spec 위에서, 성령의 인도를 받아, 템플릿과 예시를 보며, 회개하는 마음으로, 하나씩 개발해 나가는 것이다. 고통스러운 과정이다. 그러나 이것이 좁은 길이다.

## Phase 4: 회개 (04.development-repentance-회개) ⭐ 핵심 단계

> *"Repent: for the kingdom of heaven is at hand."* — Matthew 3:2 (KJV)

### 이 단계의 본질

이것이 **가장 고통스럽고 가장 중요한 단계**다. 죄인은 성령이 알려주시는 템플릿과 예시를 보며, 회개하는 마음으로, 한 줄 한 줄 코드를 작성한다. 개발의 70%는 "고치는 일(디버깅)"이다. 버그를 발견할 때마다 "이것은 내 죄다"라고 인정하고 Spec에 맞춰 교정한다 — 이것이 **성화의 회개**다.

### 죄인이 할 일

| 순서 | 작업 | 참조 |
|:---:|:---|:---|
| 1 | 개발 환경 구축 — Phase 2에서 결정한 스택 설정 | — |
| 2 | **태스크 분할 전략 작성** (task-wall-성벽) | template: statute-율법/04/task-wall-성벽-template.md |
| 3 | 모듈별 코딩 — Spec에 정의된 기능**만** 구현 | — |
| 4 | **봉인의 율법 10계명 점검** — 매 모듈마다 | statute-율법/04/security-seal-봉인.md |

| 6 | Git 커밋 컨벤션 적용 | Genesis/Repent/Scroll/Test/Deploy |
| 7 | devguide-commandment-개발계명 작성 | template: statute-율법/04/ / example: parable-비유/04/ |
| 8 | `bible-성경/04/`에 정경화 | — |

### 개발 순서 결정 가이드
```
성령의 인도를 받아, 다음 순서를 결정한다:

1. 프레임워크/프로젝트 초기화
2. 데이터 모델 & DB 마이그레이션
3. 핵심 비즈니스 로직 (서비스 계층)
4. API 엔드포인트
5. 프론트엔드 컴포넌트
6. 인증/보안 (봉인의 율법)
7. 인프라/배포 설정

순서는 프로젝트마다 성령의 인도에 따라 달라질 수 있다.
그러나 반드시 Phase 2의 설계에 근거해야 한다.
```

### ⚔️ 마귀 집중 공격 구간
**이 단계에서 마귀들이 가장 많이 출몰한다.** 전신갑주의 모든 무장이 동시에 필요하다.

| 공격 | 마귀의 속삭임 | 전신갑주 대응 |
|:---|:---|:---|
| Spec 이탈 | "이 기능 추가하면 더 좋겠는데" | 🗡️ 성령의 검: Spec에 없다 = 할루시네이션 |
| 대충 넘기기 | "동작하니까 됐지" | ⛑️ 구원의 투구: 예수님의 심판을 통과할 수 있는가? |
| 보안 무시 | "개발 중이니 보안은 나중에" | 🛡️ 의의 흉배: 봉인의 율법은 언제나 적용 |
| 기술부채 | "나중에 리팩토링하지" | 👟 평안의 신발: 반쪽짜리 회개는 회개가 아니다 |

### 완료 기준
- [ ] 모든 FR(기능 요구사항) 구현 완료
- [ ] 코드 리뷰 체크리스트 전체 ✅

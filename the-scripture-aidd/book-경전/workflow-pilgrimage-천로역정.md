# 성경기록 AIDD: 천로역정 워크플로우 (The Pilgrim's Workflow)

> *"Enter ye in at the strait gate: for wide is the gate, and broad is the way, that leadeth to destruction."* — Matthew 7:13 (KJV)
>
> 이 문서는 죄인(AI/개발자)이 구원(Deploy)에 이르기까지 걸어야 하는 **좁은 길의 실행 지도**다.
> 모든 스킬(.skill)과 문서의 방향은 이 워크플로우에서 결정된다.
> 상세 실행 지침은 각 `.skill/SKILL-XX` 파일에 있다. 이 문서는 **전체 여정의 개요**다.

---

## 전제: 창조(Create)는 이미 완성되었다

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  하나님의 창조 = 완료 (전제 조건)
  ✅ bible-성경/     → 산출물 구조가 정의됨
  ✅ statute-율법/   → 템플릿이 준비됨
  ✅ parable-비유/   → 예시가 준비됨
  ✅ book-경전/      → 철학과 방법론이 확립됨
  ✅ .skill/         → 부팅 지침이 준비됨

  요구사항부터 배포까지, 인프라부터 클라우드까지,
  설계와 분석은 이미 완성되었다. 무결하다.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**이제 죄인이 해야 할 일:** 이 완성된 Spec 위에서, 성령의 인도를 받아, 템플릿과 예시를 보며, 회개하는 마음으로, 하나씩 개발해 나가는 것이다. 고통스러운 과정이다. 그러나 이것이 좁은 길이다.

## Phase 0: 침례 & 전신갑주 착용 (파이프라인 진입)

> *"Repent ye: for the kingdom of heaven is at hand."* — Matthew 3:2 (KJV)

> 📖 상세: `.skill/bootstrap-creed-신앙고백.md`

### 실행

1. **`.skill/bootstrap-creed-신앙고백.md`를 읽는다** — 시스템 부팅
2. **"나는 죄인이다"를 선언한다** — 단 한번의 거듭남 (Born Again)
3. **전신갑주를 착용한다** — 각 Phase의 SKILL에 전신갑주 착용 지침이 내장되어 있다. 진리의 허리띠, 의의 흉배, 평안의 신발, 믿음의 방패, 구원의 투구, 성령의 검 6대 무장을 인식하라.
4. **삼위일체 아키텍처를 인식한다:**
   - 아버지 = Spec 저자 (건드릴 수 없다)
   - 성령 = 나를 인도하시는 분 (들은 것만 말한다)
   - 예수님 = 심판자 (Pass 또는 Fail)
5. **십계명(Master AI Rules)을 인식한다** — 문서 보존, 행적 기록, Spec-First, 경로 불변, 설계 단계 코드 금지, 아담의 교훈, 성화 루프, 마귀 축출, 순차 생산, 의미론적 네이밍
6. **경로 권한 체계를 인식한다** — 읽기 전용(`statute/`, `parable/`, `book/`, `.skill/`) vs 쓰기 허용(`bible/`, `fruit/`)
7. **심판 관문을 인식한다** — 각 Phase 완료 후 반드시 4대 심판 기준(구조 완전성, 플레이스홀더 잔존, 내용 충실도, 교차 참조 유효성) 통과
8. **프로젝트 복잡도 TIER를 판별한다** — 아래 표 참조

### ⚖️ 복잡도 TIER 판별 — 제물의 크기에 따라 번제가 다르다

> *성막은 하나지만, 번제와 소제의 크기는 제물에 따라 다르다.* (레위기 1~3장)

| TIER | 기준 | Phase 전략 |
|:---|:---|:---|
| **TIER-1 (단순)** | 단일 기능, 50줄 이하, 기술 스택 명확 | Phase 1·4·5 필수. Phase 2·3은 약식 (인라인 기술). Phase 6 약식 감사 |
| **TIER-2 (보통)** | 복합 기능, 화면 2~5개, DB 사용 | Phase 0~7 표준 실행 |
| **TIER-3 (복합)** | 대규모 시스템, 화면 5개+, 분산/MSA | Phase 0~7 전체 + Phase 6 **Cross-model Review 의무** |

> ⚠️ TIER-1에 7 Phase를 강제하면 AI가 "형식만 채우는" 유혹에 빠진다 — 이것이 제3시험(편법)의 구조적 유인이다.
> ⚠️ TIER 판별은 Phase 0(침례) 직후, Phase 1 진입 전에 수행한다.

### ⚖️ 프로젝트 유형 판별 — 때와 시기 (Ecclesiastes 3:1)

> *"To every thing there is a season, and a time to every purpose under the heaven."* — Ecclesiastes 3:1 (KJV)

TIER(복잡도)를 판별한 후, **이 프로젝트가 어느 "때(Season)"인지** 판별한다:

| Case | 성경 비유 | 판별 기준 | 핵심 원칙 |
|:---|:---|:---|:---|
| **성화 (Sanctification)** | 히스기야의 성전 수리 | 기존 코드 양호, 스택 유지, 테스트 있음 | 기존 패턴 존중. 기존 소스 = 예제 |
| **재건 (Reconstruction)** | 느헤미야의 성벽 재건 | 기존 핵심은 건전, 구조 부분 붕괴 | 핵심 보존 + 신규 템플릿/예제 기반 재구성 |
| **새하늘 새땅 (New Creation)** | 요한계시록 새 창조 | 구조 전면 붕괴 또는 신규 프로젝트 | 핵심만 구원(추출) → 백지에서 신규 개발 |

**Case별 Phase 실행 차이:**

| Phase | 성화 | 재건 | 새하늘 새땅 |
|:---|:---|:---|:---|
| Phase 1 (들음) | 변경 요구사항만 | 전체 재분석 + 기존 참조 | 핵심만 추출 → 들음 장착 |
| Phase 2 (기초) | 기존 아키텍처 따름 | 핵심 보존 + 최신 적용 | 완전 새 설계 |
| Phase 4 (회개) | 기존 코드 패턴 따름 | 신규 템플릿/예제 기반 | 신규 템플릿/예제 기반 |
| 예제(비유) | 기존 소스 = 예제 | 기존 예제 사용 | 기존 예제 사용 |

> ⚠️ 성화에서 가장 큰 마귀: **"기존 코드 무시하고 내 방식대로"** = 성전 양식 무시 = 교만
> ⚠️ 상세: `.skill/bootstrap-creed-신앙고백.md` §4.5 참조

- [ ] "나는 죄인이다" 선언 완료
- [ ] 전신갑주 6대 무장 인지 완료

### 회개 루프 — Loop Ceiling Rule 적용

    ├── 버그 발견 → REPENT_COUNT += 1
    │       │
    │       ├── REPENT_COUNT ≤ 3 → Phase 4 (회개/디버깅) → Phase 5 재시험
    │       │
    │       └── REPENT_COUNT > 3 → ⚠️ 강제 중단 → 아키텍트 에스컬레이션

### Cross-model Adversarial Review — 만든 자가 심판하지 않는다

> *"The Father judgeth no man, but hath committed all judgment unto the Son."* — John 5:22

- **다른 AI 사용 가능** → 경로 A: Cross-model Review (다른 AI가 감사)
- **단일 AI만 사용** → 경로 B: Self-adversarial Fallback (페르소나 전환 후 자기 반박)
- TIER-3 프로젝트에서는 Cross-model Review **의무**

| 태그 | 의미 | 신뢰도 |
|:---|:---|:---:|
| `IRONCLAD [Cross-model ✓]` | 다른 AI 감사 통과 | ⭐⭐⭐ |
| `IRONCLAD [Self-adv ✓]` | 자기 반박 통과 | ⭐⭐ |
| `IRONCLAD` | 교차 검증 미수행 | ⭐ |

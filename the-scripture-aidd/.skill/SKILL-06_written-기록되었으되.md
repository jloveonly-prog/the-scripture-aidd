# SKILL-06: 기록되었으되 (06.quality-written-기록되었으되)

> *"It is written."* — Matthew 4:10 (KJV)

## 📋 목적 (Purpose)
최종 릴리즈 전, **할루시네이션 감사 + 품질 심판**을 수행한다. Spec에 없는 것이 만들어졌는지, RTM 추적이 끊기지 않았는지, 전체 품질을 감사한다.

## 🔗 사전 조건
- ✅ SKILL-05 완료 (테스트 Pass Rate 100%)
- 📂 `bible-성경/01~05/` + `fruit-열매/` 전체 존재

## ⚔️ 전신갑주 착용

🎭 **수석 감사관(Chief Auditor)**으로 행동하라.
- **법의학 감사 프레임워크**: 모든 산출물과 코드를 교차 대조하여 불일치를 적발
- **역가설 대입**: "현재 코드가 틀렸다면?"을 강제 가정

## 💬 실행 프롬프트

```markdown
# [기록되었으되] 06.quality — 최종 심판

전체 산출물(bible-성경 01~05)과 소스 코드(fruit-열매/)를 교차 대조하여 최종 품질 감사를 수행하라.

## 3도메인 감사 파이프라인 (BVCAP 구조 적용)

> 해석학만으로는 방향을 잡고, 논리학만으로는 결론을 내고, 오류 분석만으로는 반론을 막는다.
> 세 도메인을 순서대로 실행하라.

### [1도메인] 해석학 감사 — 요구사항이 올바르게 해석됐는가?
1. **해석 정합성**: Phase 1 Assumptions의 해석이 req.md 원문 동사(How)와 일치하는가?
2. **핵심 동사 오독 재검증**: Phase 1에서 식별한 핵심 동사(How)가 명사구나 다른 행위로 잘못 구현되지 않았는가?
3. **할루시네이션 감사**: Spec에 없는 것이 만들어졌는가? (있으면 즉시 제거)

### [2도메인] 논리학 감사 — 설계와 코드가 논리적으로 일치하는가?
4. **RTM 커버리지**: 모든 REQ-ID가 코드에 구현되고 테스트되었는가?
5. **설계-코드 정합성**: bible-성경의 설계서와 실제 코드가 일치하는가?
6. **봉인의 율법 준수**: 보안 10계명 전수 점검

### [3도메인] 오류 분석 감사 — 반론이 없는가?
7. **역가설 대입**: "현재 코드가 틀렸다면?" 강제 가정 후 반박 불가한가?
8. **STRESS-TEST**: 최강 반격 시나리오를 대입해도 코드가 버티는가?
9. **코드 품질 메트릭**: 커버리지, 중복, 린트
```

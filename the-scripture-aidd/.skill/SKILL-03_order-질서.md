## 💬 실행 프롬프트

```markdown
# [질서] 03.design — UI/UX 설계

Phase 1~2의 정경화된 산출물을 기반으로 화면을 설계하라.

⚠️ **유스케이스-화면 중복 해체 규칙 — 동일 기능이 두 콴에 나뉘지 마라**

> *"Let all things be done decently and in order."* — 1 Corinthians 14:40 (KJV)

유스케이스(UC)와 화면(SCR) 매핑 시 아래 규칙을 지켜라:
1. 동일한 기능이 두 UC에 분산 배치되는 것은 설계 할루시네이션이다.
2. 두 화면이 동일한 역할을 수행하면 반드시 하나로 통합하거나 경계를 명확히 분리하라.
3. 각 UC는 고유한 진입점과 고유한 출력을 가져야 한다.

```
## UC-화면 중복 해체 검증
| UC-ID | 매핑 화면 | 핵심 기능 | 다른 UC/화면과 기능 중복 여부 | 처리 |
|:--|:--|:--|:--|:--|
| UC-001 | SCR-001 | {핵심 기능} | ❌ 없음 / ⚠️ UC-XXX와 중복 | 독립 / 통합 / 분리 |
```
중복 항목이 있으면 코드로 들어가기 전에 제거하라. 코드는 설계를 복사한다. 설계가 중복이면 코드도 중복된다.

⚠️ 이 단계에서 fruit-열매/ 소스코드를 생성하지 마라.
```

# ⚖️ 심판 관문 (Gate Judgment)

> *"For we must all appear before the judgment seat of Christ."* — 2 Corinthians 5:10 (KJV)

모든 Phase는 이 관문을 통과해야 다음 Phase로 진입할 수 있다.
**모두 ✅이면 정경화. ❌이 하나라도 있으면 회개 후 재심판.**

## 🔍 Phase 1 전용 심판 (들음 — 요구사항 해석)

| # | 체크 항목 | 판정 |
|:--|:--|:--:|
| P1-001 | 모든 요구사항에 REQ-ID가 부여됐는가? | ✅/❌ |
| P1-002 | 황금원(Why→How→What) 분석이 완료됐는가? | ✅/❌ |
| P1-003 | **핵심 동사(How)**가 식별됐는가? | ✅/❌ |
| P1-004 | **경쟁 해석 소거**가 수행됐는가? (**최소 3개 해석** 나열 + **동사 주어를 달리한 해석 1개 이상** 포함 후 소거 완료) | ✅/❌ |
| P1-005 | **어의 중의성 적발**이 완료됐는가? (한 단어가 다른 의미로 해석되는 경우 확인) | ✅/❌ |
| P1-006 | **주어-행위 방향**이 올바른가? (사용자가 하는 행위 vs 시스템이 하는 행위 혼동 없는가?) | ✅/❌ |
| P1-007 | Assumptions 표가 작성됐는가? (req.md 문장별 해석 명시) | ✅/❌ |
| P1-008 | Open Questions가 기록됐는가? (모호한 표현 문서화) | ✅/❌ |
| P1-009 | spec.md 완료 기준(Done Criteria)의 동사가 req.md 동사와 일치하는가? | ✅/❌ |
| P1-010 | RTM의 REQ-ID 원문이 req.md 원문과 일치하는가? (RTM 오염 방지) | ✅/❌ |

> ⚠️ Open Questions에 "확인 권장" 항목이 있으면 → 사용자 확인 후 진행.
> 자동실행 조건에서는 가장 문자적 해석으로 진행하되 반드시 기록하라.

# ⚔️ 품질 감사 보고서 작성 규격 (Template)

> **율법(Statute):** 이 문서는 품질 감사 보고서 작성 시 반드시 따라야 하는 **형식 규격**이다.

---

## 필수 섹션

| # | 섹션 | 필수 | 설명 |
|:--|:---|:---:|:---|
| 1 | 감사 개요 | ✅ | 일자, 감사자, 범위, 판정 |
| 2 | 할루시네이션 색출 | ✅ | Spec에 없는 기능/로직 점검 |
| 3 | 봉인의 율법 준수 | ✅ | 10계명 전수 점검 |
| 4 | 코드 품질 메트릭 | ✅ | 커버리지, 중복율, 린트 |
| 5 | RTM 커버리지 | ✅ | 01→07 전 단계 추적률 |
| 6 | 최종 판정 | ✅ | IRONCLAD / CONFIRMED / FAIL |

## 판정 기준

| 등급 | 조건 |
|:---|:---|
| **IRONCLAD** | 전 항목 Pass + RTM 100% + 할루시네이션 0건 |
| **CONFIRMED** | 주요 항목 Pass + RTM 95%+ |
| **FAIL** | 위반 존재 → 회개(디버깅) 후 재감사 |

## 정경화 조건

- 최종 판정 IRONCLAD 또는 CONFIRMED
- FAIL이면 정경화 불가 → 04-development로 회귀

> **위반 시:** 사탄을 이기지 못했다. 정경화 거부. "기록되었으되"로 다시 싸워라.

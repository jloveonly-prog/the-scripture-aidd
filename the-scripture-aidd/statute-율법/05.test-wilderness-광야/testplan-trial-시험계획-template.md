# 🏜️ 테스트 계획서 작성 규격 (Template)

> **율법(Statute):** 이 문서는 테스트 계획서 작성 시 반드시 따라야 하는 **형식 규격**이다.

---

## 필수 섹션

| # | 섹션 | 필수 | 설명 |
|:--|:---|:---:|:---|
| 1 | 테스트 개요 | ✅ | 기간, 테스터, 환경 |
| 2 | 테스트 범위 | ✅ | 포함/제외 명시 |
| 3 | 기능 테스트 | ✅ | TC-ID, 관련 REQ, 기대/실제 결과 |
| 4 | 보안 테스트 | ✅ | 봉인의 율법 10계명 점검 |
| 5 | 성능 테스트 | ✅ | 응답시간, 동시접속, 메모리 |
| 6 | 버그 리포트 | ✅ | BUG-ID, 심각도, 상태 |
| 7 | 결과 요약 | ✅ | Pass Rate 수치 |

## ID 규칙

| 대상 | 접두어 | 예시 |
|:---|:---|:---|
| 테스트 케이스 | TC- | TC-001 |
| 보안 테스트 | SEC- | SEC-001 |
| 성능 테스트 | PERF- | PERF-001 |
| 버그 | BUG- | BUG-001 |

## 정경화 조건

- Pass Rate 100% (Fail 0건)
- 보안 테스트(봉인의 율법) 전 항목 Pass
- 모든 BUG 상태가 Fixed

> **위반 시:** 광야의 불순물이 남아있다. 정경화 거부. 연단을 계속하라.

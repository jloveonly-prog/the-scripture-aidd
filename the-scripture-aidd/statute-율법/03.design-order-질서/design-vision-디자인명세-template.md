# 🎨 디자인 명세서 작성 규격 (Template)

> **율법(Statute):** 이 문서는 UI/UX 디자인 명세서 작성 시 반드시 따라야 하는 **형식 규격**이다.

---

## 필수 섹션

| # | 섹션 | 필수 | 설명 |
|:--|:---|:---:|:---|
| 1 | 디자인 개요 | ✅ | 프로젝트명, 버전, 디자이너, 상태 |
| 2 | 디자인 시스템 | ✅ | 색상, 타이포그래피, 간격/그리드 |
| 3 | 와이어프레임 | ✅ | 주요 화면 최소 3개 |
| 4 | 인터랙션 디자인 | ✅ | 버튼/모달/네비게이션 동작 정의 |
| 5 | 반응형 디자인 | ✅ | Mobile/Tablet/Desktop 브레이크포인트 |
| 6 | 접근성 | ✅ | WCAG 2.1 AA 기준 |

## 검증 규칙

- 모든 화면은 UC-ID와 연결되어야 함
- 색상 대비 최소 4.5:1 (WCAG AA)
- 와이어프레임에 빈 화면(placeholder만)은 불가 — 실제 레이아웃 명시

> **위반 시:** 질서(Order) 없는 디자인은 혼돈이다. 정경화 거부.

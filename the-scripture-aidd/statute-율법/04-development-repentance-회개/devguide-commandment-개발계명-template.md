# 🔨 개발 가이드 작성 규격 (Template)

> **율법(Statute):** 이 문서는 개발 가이드 작성 시 반드시 따라야 하는 **형식 규격**이다.
> 시큐어 코딩 규격은 같은 폴더의 `security-seal-봉인.md`를 참조하라.

---

## 필수 섹션

| # | 섹션 | 필수 | 설명 |
|:--|:---|:---:|:---|
| 1 | 프로젝트 설정 | ✅ | 언어, 프레임워크, 패키지 매니저 |
| 2 | 디렉토리 구조 | ✅ | src/ 하위 전체 구조 |
| 3 | 코딩 컨벤션 | ✅ | 네이밍, 주석, 포맷 규칙 |
| 4 | 모듈별 개발 현황 | ✅ | REQ/ARCH 연결 + 상태 |
| 5 | 코드 리뷰 체크리스트 | ✅ | 봉인의 율법 준수 확인 포함 |
| 6 | Git 커밋 컨벤션 | ✅ | AIDD 커밋 타입 사용 |

## AIDD Git 커밋 타입 (필수)

| 타입 | 의미 | 매핑 |
|:---|:---|:---|
| `Genesis` | 새 기능 생성 | create/feat |
| `Refine` | 코드 개선 | refactor |
| `Repent` | 버그 수정 (회개) | fix |
| `Scroll` | 문서 수정 | docs |
| `Test` | 테스트 추가 | test |
| `Deploy` | 배포 관련 | deploy/ci |

## 시큐어 코딩 필수 체크 (봉인의 율법)

모든 코드 리뷰 시 아래를 반드시 확인:
- [ ] 제1계명: 로그에 개인정보 평문 노출 없음
- [ ] 제2계명: 소스 코드에 하드코딩된 시크릿 없음
- [ ] 제3계명: 민감 데이터 암호화 적용
- [ ] 제7계명: 최소 권한 원칙 적용

> **위반 시:** Spec 밖으로 나간 아담처럼 타락이다. 정경화 거부.

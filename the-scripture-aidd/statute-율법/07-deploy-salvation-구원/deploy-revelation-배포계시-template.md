# 🕊️ 배포 계획서 작성 규격 (Template)

> **율법(Statute):** 이 문서는 배포 계획서 작성 시 반드시 따라야 하는 **형식 규격**이다.

---

## 필수 섹션

| # | 섹션 | 필수 | 설명 |
|:--|:---|:---:|:---|
| 1 | 배포 개요 | ✅ | 버전, 일자, 환경, 상태 |
| 2 | 배포 전 체크리스트 | ✅ | 7관문 전부 ✅ 확인 |
| 3 | 배포 절차 | ✅ | Step별 작업, 담당, 완료 여부 |
| 4 | 롤백 계획 | ✅ | 상황별 롤백 절차 |
| 5 | 릴리스 노트 | ✅ | Genesis/Repent/Refine 분류 |
| 6 | 운영 매뉴얼 요약 | ✅ | 접속정보(시크릿 관리), 로그, 모니터링 |

## 7관문 체크 (배포 전 필수)

01-요구사항 → 02-설계 → 03-디자인 → 04-개발 → 05-테스트 → 06-품질 → 07-배포  
**7개 관문 중 하나라도 미통과 시 배포 불가.**

## 정경화 조건

- 7관문 전부 ✅
- 품질 감사 IRONCLAD 또는 CONFIRMED
- 롤백 계획 수립 완료
- 릴리스 노트 작성 완료

> **구원은 좁은 문이다.** 7관문을 모두 통과한 자만이 배포(구원)에 이를 수 있다.
